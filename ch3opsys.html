Operating Systems Terminology

TERM

DEFINITION

operating system

The softare that controls the overall operation of the computer including file management, the user interface, and the program execution environment.

job queue

In batch processing systems, a job queue is defined as the jobs residing in mass storage waiting for execution as a type of storage organization for objects.

FIFO

the objects are removed from the queue in the order in which they arrived.

interactive processing

allows a program being executed to carry on a dialogue with the user through remote terminals

time-sharing

refers to operating systems that provided service to multiple users at the same time

multiprogramming

time is divided into intervals and then the execution of each job is restricted to only one interval at a time.

application software

Application software consists of the programs for performing tasks particular to the machine's utilization.

system software

system software performs those tasks that are common to computer systems in general

utlity software

The majority of an installation's utility software consists of programs for performing activities that are fundamental to computer installations but not included in the operating system.

graphical user interace (GUI)

objects to be manipulated, such as files and programs, are represented pictorially on the display as icons. These systems allow users to issue commands by using one of several common input devices

shell

user interfaces that communicat with users through textual messages using a keyboard and monitor screen.

kernel

An operating system's kernel contains those software components that perform the very basic functions required by the computer installation.

device driver

the software unit that communicates with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine.

file manager

coordinate the use of the machine's mass storage facilities. More precisely, the file manager maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files.

memory manager

component of an operating system's kernel, charged with the task of coordinating the machine's use of main memory

virtual memory

large “fictional” memory space created by paging

boot strapping

the procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory

ROM

read-only memory, because its contents can be read but not altered.

deadlock

the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.

spooling

the operating system makes the nonshareable resource appear shareable by creating the illusion of more than one printer, in order to hold data for output at a later but more convenient time

super user (or administrator)

can alter settings within the operating system, modify critical software packages, adjust the privileges granted to other users, and perform a variety of other maintenance activities that are denied normal users.

multi-factor authentication

the user must provide a password, and also a special access code text messaged to the user's smartphone during the login process. A third set of authentication factors might depend on something related to who the user is
