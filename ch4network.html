Operating Systems Terminology

TERM

DEFINITION

Network Classifications

    1. LAN
    2. MAN
    3. WAN

    1. Local Area Network
    2. Metropolitan Area Network
    3. Wide Area Network

Network models - etc

    1. bus network
    2. star network

    1. A bus networok is created by running links from each computer to a central location where they are connected to a device called a hub.
    2. Machines in the network communicate through an intermediary central machine (e.g. a switch.)

Protocols - etc

    1. CSMA/CD Ethernet
    2. CSMA/CA Wifi

    1. Carrier Sense, Multiple Access with Collision Detection (CSMA/CD)
    2. Carrier Sense, Multiple Access with Collision Avoidance (CSMA/CA)

Combining networks

    1. repeater
    2. bridge
    3. switch
    4. router
    5. gateway

    1. a device that passes signals back and forth between the two original buses (usually with some form of amplification) without considering the meaning of the signals
    2. connects two buses, but it does not necessarily pass all messages across the connection. Instead, it looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.
    3. a bridge with multiple connections, allowing it to connect several buses rather than just two
    4. their purpose is to forward messages in their proper directions. This forwarding process is based on an internet-wide addressing system in which all the devices in an internet (including the machines in the original networks and the routers) are assigned unique addresses
    5. The "point" at which one network is linked to an internet

Client/server model

    1. P2P: peer-to-peer
    2. distributed system
    3. cloud computing

    1. involves processes that provide ser vice to and receive service from each other
    2. consist of software units that execute as processes on different computers
    3. huge pools of shared computers on the network can be allocated for use by clients as needed

Internet Addressing

    1. Tier 1
    2. Tier 2
    3. Tier 3

    1. ISPs that consist of very high-speed, high-capacity, international WANs. These networks are thought of as the backbone of the Internet.
    2. ISPs that tend to be more regional in scope and less potent in their capabilities.
    3. Tier 3 (or access ISP) is essentially an independent Internet, sometimes called an intranet, operated by a single authority that is in the business of supplying Internet access to individual homes and busi nesses.

IP address

    1. ICANN
    2. DNS
    3. TLD: Top-level domain
    4. Country-code TLDs

    1. Internet Corporation for Assigned Names and Numbers (ICANN), which is a non profit corporation established to coordinate the Internet's operation. See: ICANN homepage
    2. Domain Name System - for conversion of mnemonic address into numeric IP
    3. Top-level domain: domain name suffixes
    4. two-letter TLDs for each specific country. See World Standard tlds

World Wide Web

    1. URL
    2. Client-side activities
    3. Server-side avtivities

    1. Uniform Resource Letter, the unique address given to each document
    2. e.g. an enduser inputs information
    3. e.g. the information is used to construct an appropriate customized webpage

Internet four-layer software

    1. application layer
    2. transport layer
    3. network layer
    4. link layer

    1. Where a message originates and is coupled with a destination address
    2. Where a message is chopped into packets
    3. Where a network assigns intermediate addresses to each packet
    4. Receives the transferred packet

Internet is:

    1. packet-switching network
    2. fault tolerant
    3. redundant

    1. through which digital data is sent by breaking the data into blocks of bits called packets, which contain both the data being transmitted and control information for routing the data
    2. The cumulative result of these features for transporting, routing, and for warding packets is an Internet that is tolerant of many types of faults, and that can fall back to other, redundant pathways for delivery when congestion or failure occur.
    3. The redundancy of routing (i.e., more than one way to route data) between two points on the Internet increases the reliability of the Internet and helps it scale to more devices and more people

Cybersecurity

    1. malware
    2. virus
    3. worm
    4. trojan horse
    5. spyware
    6. phishing
    7. DDoS attack

    1. malicious software
    2. software that infects a computer by inserting itself into pro grams that already reside in the machine.
    3. an autonomous program that transfers itself through a net work, taking up residence in computers and forwarding copies of itself to other computers.
    4. a program that enters a computer system disguised as a desirable program, such as a game or useful utility package, that is willingly imported by the victim.
    5. software that collects information about activities at the computer on which it resides and reports that information back to the insti gator of the attack.
    6. a technique of obtaining information explicitly by simply asking for it. The term phishing is a play on the word fishing because the process involved is to cast numerous “lines” in hopes that someone will “take the bait.”
    7. distributed denial of service (DDoS), the process of overloading a computer with messages, when the work of generating the attacks is distributed over many hosts

Protection

    1. firewall
    2. spam filter
    3. proxy server
    4. antivirus software
    5. cryptography
    6. public key encryption

    1. primary prevention software for filtering traffic passing through a point in the network
    2. firewalls designed to block unwanted email.
    3. software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
    4. used to detect and remove the presence of known viruses and other infections.
    5. the study of sending and receiving messages securely in the presence of adversaries
    6. techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
