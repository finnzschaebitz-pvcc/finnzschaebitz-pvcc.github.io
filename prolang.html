Programming Languages

TERM

DEFINITION

machine language

the collection of bit patterns that serve as instructions for a CPU

debugging

locating and correcting errors

assembler

the objects are removed from the queue in the order in which they arrived.

assembly language

built upon the machine language, it is a mnemonic system for representing programs

translator

akin to assembly language, it translates programs expressed in the high-level primitives of third-generation programming languages

compiler

a translator that compiles the machine instructions into short sequences to simulate the activity requested by a single high-level primitive.

interpreter

Like translators, except that they execute the instructions as they are translated instead of recording the translated version for future use.

cross-platform software

software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.

natural language

a language which evolved over time without formal grammatical analysis

formal language

languages with precisely defined grammars

machine independence

the ability of a third-generation program to run on any machine, as well as the possibility of humans communicating with machines in terms of abstract concepts rather than forcing them to translate these concepts into machine-compatible form

high-level programming language

igh-level programming languages provide more abstractions for the programmer and make it easier for people to read and write a program

object-oriented programming (OOP)

Following this paradigm, a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand

scripting language

A subset of the imperative programming languages, used to perform administrative tasks rather than to develop complex programs. The expression of such a task is known as a script

data types:

  • integer
  • float
  • string
  • numeric data consisting of whole numbers, probably stored using two's complement notation
  • numeric data that might contain values other than whole numbers, probably stored in floating-point notation
  • a sequence of characters that can contain letters, numbers, symbols and even spaces

primitive data type

The data types that are included as primitives in a programming language, such as 'int' for integer and 'char' for character

data structure

the conceptual shape or arrangement of data

Python collection data structures

  • list
  • tuple
  • set
  • dictionary
  • Lists are used to store multiple items in a single variable, and are created using square brackets
  • Tuples are used to store multiple ordered and unchangeable items in a single variable, and are written with round brackets
  • Sets are used to store multiple unordered, mostly unchangeable, and unindexed items in a single variable, and are written with curly brackets
  • Dictionaries are used to store ordered, changeable, and dissimilar data values, are written with curly brackets, and have keys and values

function

a set of instructions for performing a task that can be used as an abstract tool by other program units

local variable

a variable declared within a function, meaning that it can be referenced only within that function

global variable

Variables whose scopes are not restricted to a particular part of a program

source program

a program in its original form

object program

a program translated from one language to another
