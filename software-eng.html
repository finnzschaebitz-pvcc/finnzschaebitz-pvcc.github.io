software engineering

systems that combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.

Association for Computing Machinery (ACM)

  • purpose of ACM
  • benefits of ACM membership
  • student ACM membership

The world's largest learned society for scientific computing. It is a non-profit and headquartered in New York.

  • its purpose is to advance computing as a science & profession, per its motto
  • benefits of an ACM membership include, but are not limited to access to publications, networking opportunities, discounts on conferences and events, career & educational resources, Special Interest Groups (SIGs), professional development, as well as access to the ACM digital library
  • student memberships are less expensive than professional membershisp, yet still provide access to many of the same resources as a professional membership, and may have specific benefits aimed at supporting an academic journey

Institute of Electrical and Electronics Engineers (IEEE)

  • purpose of IEEE
  • benefits of IEEE membership
  • student IEEE membership

The IEEE is a nonprofit based in New York, which concerns itself with electronics engineering, electrical engineering, and other related disciplines.

  • Per its homepage, the "IEEE and its members inspire a global community through its highly cited publications, conferences, technology standards, and professional and educational activities."
  • IEEE members are presented with resources, such as the IEEE Xplore Digital Library, opportunities to network through technical communities, as well as discounts to help advance their careers.
  • Student memberships are garnered toward people who need access to resources that are relevant to their educational journey, such as access to funds including travel grants, scholarships, awards etc.

Software Life Cycle: Development phase

  • Requirements analysis
  • Design
  • Implementation
  • Testing

The major steps in the traditional software development life cycle are requirements analysis, design, implementation, and testing

  • done in order to specify what services the proposed system will provide, to identify any conditions (time constraints, security, and so on) on those services, and to define how the outside world will interact with the system
  • involves creating a plan for the construction of the proposed system and developing a solution to the problem
  • n involves the actual writing of programs, creation of data files, and development of databases
  • formerly constricted to debugging programs and confirming the final software compatability. Now viewed as an integrated process to incorporate within the other steps

modular implementation

Determining whether modules will serve as functions or objects within the software design process, and then implementing it as such

Unified Modeling Language (UML)

a modern collection of tools that has been developed with the object-oriented paradigm in mind

use case

interactions between the system and its users

use case diagram

a tool which attempts to capture the image of the proposed system from the user's point of view

Data relationships:

  • one-to-one
  • one-to-many
  • many-to-many

The three basic forms by which association multiplicities occur

  • exemplified by the association between patients and occupied private rooms in that each patient is associated with only one room and each room is associated with only one patient
  • exemplified by the association between physicians and patients in that one physician is associated with many patients and each patient is associated with one (primary) physician.
  • would occur if we included consulting physicians in the physician-patient relationship

design pattern

a predeveloped model for solving a recurring problem in software design

software testing:

  • glass box testing
  • black box testing
  • beta testing
  • alpha testing

identifying problematic modules and testing them more thoroughly

  • the software tester is aware of the interior structure of the software and uses this knowledge when designing the test
  • refers to tests that do not rely on knowledge of the software's interior composition. In short, black-box testing is performed from the user's point of view
  • a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market
  • Similar testing performed at the developer's site

Documentation:

  • user documentation
  • system documentation
  • technical documentation

an important part of a final software package, by which users learn to use and maintain it

  • done to explain the features of the software and describe how to use them
  • done to escribe the software's internal composition so that the software can be maintained later in its life cycle
  • done to describe how a software system should be installed and serviced (such as adjusting operating parameters, installing updates, and reporting problems back to the software's developer)

Intellectual property law:

  • software license
  • software disclaimer
  • best defense against software liability claims

In terms of software: Legal efforts to provide ownership of that software

  • whereby the rights of the developer are formally expressed
  • written by developers to protect themselves against liability
  • to apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
